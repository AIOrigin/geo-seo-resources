# Reference GEO Audit Sample

Public-safe sample summary based on an **illustrative** reference documentation site (`example.org`). Replace the domain with your own when running a real audit.

## Scope

- target site: `https://www.example.org`
- audit style: lightweight GEO snapshot
- focus: `llms.txt`, information architecture, and page citability

## Headline Findings

- `llms.txt` exists and is format-valid
- the site exposes a broad AI-readable route map
- the guides hub and prompt-template hub make good top-level discovery entry points
- the overview guide is currently more citable than the advanced prompt-template detail page

| prompt-template citability | `34.0 / 100` |

## Interpretation

### Strengths

- The site already has a machine-readable entry point.
- Hubs and detail pages are organized clearly enough for external reference.
- The route mix supports guides, prompts, prompt templates, and legal/boundary pages.

### Weak Spots

- Citability remains moderate-to-low rather than strong.
- Prompt-template pages still need richer, more quotable answer blocks.
- Some content blocks are too short or too navigation-heavy to be strong AI citation candidates.

## Recommended Promotion Routes

- [Guides hub](https://www.example.org/guides)
- [Prompt templates hub](https://www.example.org/prompt-templates)
- [Overview guide](https://www.example.org/guides/product-overview)
- [Advanced prompt techniques](https://www.example.org/prompt-templates/advanced-prompt-tips)

## Recommended Next Improvements

1. Strengthen direct-answer blocks on prompt-template pages.
2. Add more publishable Input/Output evidence to prompt-template routes.
3. Turn important operational guidance into cleaner quotable sections.
4. Keep `llms.txt` aligned as new route families expand.

## Boundary Note

This sample describes a generic third-party information site pattern. It should not be read as an official product audit for any specific vendor.
