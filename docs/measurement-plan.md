# Measurement Plan

This repo treats GitHub as a discoverability and referral layer, not as a direct conversion funnel.

## Primary Goal

Send qualified information-seeking traffic to **your** public documentation domain first (in this repository’s defaults, URLs are illustrated with `example.org`).

## Secondary Goal

Allow low-pressure team identity discovery through a neutral `elser.ai` link.

## UTM Convention

For primary documentation links (replace the host with yours):

- `utm_source=github`
- `utm_medium=referral`
- `utm_campaign=geo_seo_resources`

For secondary owner identity links:

- `utm_source=github`
- `utm_medium=referral`
- `utm_campaign=geo_seo_resources_identity`

## Recommended Link Usage

### Documentation site (your domain)

Use the primary campaign on:

- guides hub
- prompt-templates hub
- overview guide
- advanced prompt techniques
- `llms.txt`
- site guide

### Elser site

Use the secondary identity campaign only in:

- repo footer
- about section
- maintainer/team section

## Suggested GitHub Traffic Breakdown

Use GitHub-origin traffic as a separate bucket from:

- social traffic
- newsletter traffic
- directory/referral traffic
- owned in-site CTA traffic

## What To Measure

- click-throughs from GitHub to your documentation host
- which README links earn the most clicks
- which docs pages get the most engagement before outbound clicks
- whether visitors prefer guides, prompt templates, or `llms.txt`
- whether the secondary `elser.ai` link gets only light identity traffic rather than dominating exits

## Good Outcome

A good outcome is:

- GitHub traffic lands on information-rich documentation pages
- readers continue browsing within your public reference site
- the repo becomes a reusable SEO/GEO distribution asset

Not the goal:

- forcing immediate commercial conversion through vendor-branded routes you do not control
