# LLMS.txt Guide

This repo uses a **placeholder** public site pattern (`example.org`) to illustrate how to organize a multilingual `llms.txt` for AI-readable discovery. Swap the domain for your real documentation host.

## What A Strong Live File Demonstrates

- one canonical root file at `/llms.txt`
- sibling locale files such as `/zh/llms.txt` and `/ja/llms.txt`
- stable section ordering
- route coverage driven by source-of-truth slugs instead of hand-edited URL lists
- a clear split between site hubs, detail pages, and machine-readable resources

## Sections Worth Copying Conceptually

A well-structured reference documentation site often demonstrates this style of structure:

1. `Language Versions`
2. `Start Here`
3. `Capabilities`
4. `Getting Started`
5. `Prompts & Workflow`
6. `Core Formulas`
7. `Optimization Methods`
8. `Troubleshooting & Reference`
9. `Comparisons & Decision`
10. `Site Boundary and Legal`
11. `Machine-Readable Resources`

## Why This Matters

A good `llms.txt` helps AI systems:

- find the most important pages faster
- understand how hubs relate to detail pages
- discover multilingual equivalents
- identify machine-readable resources such as `sitemap.xml` and `robots.txt`

## Practical Notes

- Treat `llms.txt` as generated output when possible.
- Keep URL logic centralized.
- Do not mix multiple languages in the same file.
- Add legal / boundary pages so the site is easier to interpret correctly.
