# Landing Capability Pages

This skill covers the reusable pattern behind capability-style landing pages.

## Use When

- You are adding a new capability slug.
- You are extending page metadata, body content, or multilingual copy for a capability page.
- You are reusing the capability-page structure in another landing-page project.

## Core Steps

1. Update the canonical capability list.
2. Add or refresh page copy for each supported language.
3. Make sure route discovery and sitemap coverage stay aligned.
4. Preserve page-level SEO fields such as title, description, and structured internal linking.

## Structural Rules

- Capability pages are list-backed pages under `/capabilities/[slug]`.
- Each page should stand on its own as an information asset.
- Multilingual parity matters for the highest-priority routes.
