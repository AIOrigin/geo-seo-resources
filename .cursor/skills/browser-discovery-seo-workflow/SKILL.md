# Browser Discovery SEO Workflow

This skill covers browser-led discovery before content or route decisions are made.

## Use When

- You need search-result evidence before choosing which routes to promote.
- You want browser-based validation instead of guessing from static page content.
- You need screenshots, route comparisons, or discoverability notes for GEO or SEO planning.

## Core Workflow

1. Define the route family, keyword set, or competitor set to inspect.
2. Collect browser evidence from search results, trend sources, and reference sites.
3. Summarize what the evidence says about discoverability, not just what a page claims.
4. Turn findings into route priorities, content briefs, or follow-up audits.

## Expected Outputs

- A route priority list
- Snapshot-backed notes on page discoverability
- Follow-up recommendations for README links, route maps, or content expansion
